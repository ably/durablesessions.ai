# Contributing to durablesessions.ai

This is an open resource documenting the emergence of Durable Sessions as an AI infrastructure layer. Contributions are welcome from anyone.

## What we're looking for

- New vendor references or implementations related to durable sessions
- Corrections to existing content or broken links
- Additional research, data points, or analysis
- Translations

## How to contribute

1. **Report an issue**: [Open an issue](https://github.com/ably/durablesessions.ai/issues) to suggest changes, flag errors, or propose new content.
2. **Submit a PR**: Fork the repo, make your changes, and open a pull request.

## Guidelines

- Keep content factual and vendor-neutral. This is a community resource, not a product page.
- Reference primary sources (blog posts, documentation, GitHub issues) rather than secondary coverage.
- When adding a vendor to the convergence or references section, include a working URL.

## Structure

The site is a simple static page:

- `index.html` — all content
- `style.css` — styling
- `script.js` — scroll animations
- `vercel.json` — deployment config

No build step. Edit the HTML directly.
